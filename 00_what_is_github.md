# 깃허브를 “이해”하는 가장 쉬운 비유

깃허브는 그냥 **온라인 폴더(창고)** 입니다.

- 내 컴퓨터에 있는 폴더(프로젝트)를
- 인터넷에 올려서
- 버전 관리하고, 공유하고, 협업할 수 있게 하는 곳

---

## 3개만 알면 거의 끝

### 1) Repository (레포)
- “프로젝트 폴더 1개”

### 2) Commit (커밋)
- “저장 + 기록”
- 언제 어떤 수정이 있었는지 남김

### 3) Push (푸시)
- 내 컴퓨터에서 한 저장 기록을 깃허브(온라인)로 올리는 것

---

## 이 레포로 하는 가장 빠른 성공 경험

- index.html 수정 → 커밋 → 푸시
- 바로 내 사이트가 업데이트됨
